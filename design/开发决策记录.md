## 一、软硬件架构决策

### 1. Pi 5 + PC 混合架构（核心决策）

Orange Pi 5 全本地跑不动 LLM + CosyVoice，采用混合方案:

```
Orange Pi 5 (机器人身上)
├── 麦克风拾音 + VAD
├── Whisper tiny (本地快速转录)
├── 电机控制 + 里程计
├── 屏幕显示 (表情动画)
└── 扬声器播放 (edge-tts 或缓存wav)
     ↓ WiFi
RTX5060 PC (放柜台后面)
├── Ollama qwen2.5:1.5b (GPU推理, <1s)
├── CosyVoice TTS (GPU, 实时)
└── ChromaDB RAG

响应时间: 2-4秒 (vs 全Pi的15-30秒)
```

### 2. Pi 5 与 ROS 2 兼容性结论

- ROS 2 Humble + Pi 5 8GB → **完全够用**
- 能跑: 底盘驱动、里程计、slam_toolbox、Nav2、传感器数据转发
- 跑不动: RViz、深度相机点云
- 推荐搭配: LD19 激光雷达（几十块）+ slam_toolbox 做 2D SLAM
- 深度相机方案需要 Hailo-8L 加速卡 (~¥400)

### 3. 软件技术栈（已确定）

| 模块 | 选型 | 运行位置 |
|------|------|----------|
| ASR | Whisper base | Pi 5 |
| LLM | Qwen2.5:1.5b | PC (GPU) |
| TTS | CosyVoice (主) / edge-tts (备) | PC (GPU) / 网络 |
| RAG | TF-IDF + cosine + ChromaDB | PC |
| 意图 | 规则引擎 + LLM 分类 (混合) | 本地 |
| 唤醒 | Porcupine "你好coco" | Pi 5 |
| 电机 | JGA25-370 ×4, TB6612FNG ×2 | Pi 5 GPIO |
| UI | PyQt5 圆形屏 800×800 | Pi 5 |

## 二、硬件选型决策

### 1. 立式 v5 vs 桌面 v4

v4: 总高 564mm → 放柜台上用
v5: 总高 900mm → 立式放地面，屏幕28°仰角

选择 v5:
- 屏幕 200mm 复用，电机复用
- 底盘从 300×310 → 340×340（稳定性）
- 机身 300mm → 600mm 高（分3段打印，每段≤220mm）
- 成本增量仅 ~¥135（多一卷PLA）

### 2. 屏幕仰角设计

- 28° 上仰
- 站立用户视线向下约45°，屏幕面接近垂直视线
- 实现: 3D打印时直接留斜槽，屏幕卡入
- 独立 STL: design/screen_mount.scad（含安装支耳+麦克风槽）

### 3. 麦克风位置

- 屏幕下方"下巴"处
- 线性4麦阵列 (ReSpeaker Mic Array v2.0)
- 约1.0-1.2m高度，正对用户嘴部

### 4. 3D打印材料

- **不是 PVC**（PVC 不能3D打印）
- PLA+ 白色: 机身外壳、屏幕框
- PETG 灰色: 底盘框架、轮毂
- SLA 树脂: 精细装饰件（可选）
